# MK5 런타임 철학과 답변 계약

이 문서는 MK5의 현재 런타임이 반드시 지켜야 하는 철학과 계약을 정리한다.

## 1. 그래프 우선
- 장기 상태의 중심은 그래프다.
- 현재 입력은 그래프 접근 키이지, 자동으로 이미 알고 있던 근거가 아니다.
- 현재 턴에서 막 생성된 노드는 grounding 근거 판정에서 제외한다.

## 2. 질문 대상 grounding 우선
- search need는 그래프 총량이 아니라 질문 대상 concept grounding 존재 여부로 판단한다.
- 질문 구조 분해가 실패했다고 해서 문장 파편을 억지로 검색 의무로 만들지 않는다.
- 질문 핵심 concept가 기존 grounded node에 없을 때만 외부 검색을 시도한다.

## 3. 검색 적용 계약
- 검색 결과가 ingest되면, 같은 턴 안에서 다시 activation / thinking / grounding 판정을 수행한다.
- 검색 결과 passage는 답변에 직접 반영될 수 있어야 한다.
- coverage 보조 판정이 실패해도, 확인된 evidence 자체가 있으면 evidence-first로 답변한다.

## 4. 답변 계약
- grounded_terms와 search evidence가 모두 없으면, 모른다고 말한다.
- 검색이 실패했거나 확인이 부족하면, 추측으로 빈칸을 메우지 않는다.
- 모델 일반지식은 evidence보다 우선할 수 없다.
- evidence에 없는 세부사항은 단정하지 않는다.

## 5. 주제 따라가기
- 답변의 주제는 그래프 전체가 아니라 사용자의 마지막 질문/요청이다.
- 자기소개, 배경 설명, 질문 본문이 섞여 있어도 최종 답변은 질문 clause를 따른다.
- 내부 구조는 답변의 근거이지, 답변의 주제가 아니다.

## 6. 다이어트 원칙
- 핫패스에는 꼭 필요한 레이어만 남긴다.
- 실패를 가리는 fallback, helper, 보조 LLM 레이어를 늘리지 않는다.
- 문제를 증상 패치로 덮기보다, 실행 경로에서 원인 레이어를 제거하거나 단순화한다.
