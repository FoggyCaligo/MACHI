## 이번 턴 직후 상태
- `ModelFeedbackService`와 `ModelEdgeAssertionService`가 ChatPipeline에 연결됨
- Activation에서 concept 2-hop 확장이 동작함
- connect_type allowlist: flow/neutral/opposite/conflict
- 비허용 connect_type은 proposed 후보로 누적됨

## 이번 주 우선순위
1. 대화 E2E에서 `model_edge_assertion` 생성/강화 로그 확인
2. assertion 결과에 대한 테스트 2종 추가
   - 신규 엣지 생성
   - 기존 엣지(kind 동일) 강화
3. proposed connect_type 누적 카운트 설계
4. 승격 정책 초안 구현 (임계치 기반)
