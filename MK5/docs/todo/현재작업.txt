# 현재작업 (단기 실행)

업데이트: 2026-04-16

## 완료(최근)
- `proposed_connect_type` 승격 정책을 가중치 기반으로 반영
  - evidence_score = support_count + trust_score + source/domain weight
- 승격 정책 E2E 회귀 테스트 보강
- `revision-purpose edge` 표준 서비스 도입 (`revision_edge_service.py`)
- `TrustManager`, `StructureRevisionService`에 revision marker edge 기록 연결

## 이번 주 우선순위
1. 구조 변경 실행을 edge-first로 한 단계 더 이동
   - 목표: deactivate/merge 실행 조건을 revision marker edge 누적 기준으로 전환
2. revision marker edge를 읽는 규칙 정교화
   - `family/type/kind` 조합별 deterministic rule 추가
3. 사용자 체감 품질 회귀 방지
   - topic/identity continuity 질문셋 추가

## 리스크
- Windows 환경에서 pytest 임시 디렉터리 권한 충돌이 간헐적으로 발생한다.
- 검색/모델/그래프를 동시에 바꾸는 턴에서는 원인 분리가 어려워진다.
