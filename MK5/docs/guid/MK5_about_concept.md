# MK5 철학과 답변 계약

## 1. 프로젝트 목표
MK5는 그럴듯한 답변을 잘 만드는 시스템보다, 시간이 지날수록 사용자와 세계를 더 구조적으로 이해하는 그래프 중심 인지 시스템을 목표로 한다.

## 2. 최우선 원칙
- 현재 입력은 접근 키이지 근거가 아니다.
- 현재 턴 이전에 존재하던 grounded graph 또는 이번 턴의 검색 근거만 답변 근거가 될 수 있다.
- 구조 분해 실패는 곧바로 답변 가능을 뜻하지 않는다.
- 검색 실패, timeout, coverage 미확정 상태에서는 빈칸을 일반상식으로 메우지 않는다.
- 모르면 모른다고 답해야 한다.

## 3. 금지 사항
- 일반상식으로 빈칸 메우기
- 검색 timeout을 모델 상식으로 덮기
- 현재 입력으로 막 생성된 노드를 기존 근거처럼 취급하기
- 근거보다 자연스러운 문장을 우선하기
- helper/fallback으로 본체 계약을 우회하기

## 4. 답변 허용 조건
강한 사실 서술은 아래 둘 중 하나를 만족할 때만 허용한다.

1. prior grounded graph
- 현재 턴 이전에 존재한 grounded node / edge / pointer
- 현재 질문 대상과 직접 연결됨
- trust / provenance 기준을 만족함

2. current-turn search grounding
- 이번 턴에 검색이 실제로 수행됨
- search ingest가 끝남
- 검색 evidence passage가 현재 질문 대상에 직접 연결됨
- 가능하면 post-search re-activation / re-thinking까지 완료됨

위 조건을 만족하지 못하면:
- 설명형 단정 금지
- 세부 사실 단정 금지
- "현재 확보된 근거만으로는 확인되지 않았다"를 우선한다.

## 5. 질문 구조 분해 원칙
- 구조 분해는 검색 보조 도구이지, 답변의 진실 판정기가 아니다.
- 질문 핵심 대상 entity와 핵심 topic을 우선해야 한다.
- 인사, 연결어, 주변 설명은 그래프에는 들어갈 수 있지만 search obligation의 중심이 아니다.
- 구조 분해 실패 시에는 답변 가능이 아니라 "search 필요성 미해결"로 보는 것이 기본이다.

## 6. 검색 적용 원칙
검색이 성공했으면 다음을 목표로 한다.
- search ingest
- post-search re-activation
- post-search re-thinking
- verbalizer는 evidence passage 우선 사용

검색 결과는 참고자료가 아니라 같은 턴 결론 형성에 다시 투입되는 근거여야 한다.

## 7. 주제 따라가기 원칙
- 답변은 그래프 전체가 아니라 현재 질문 주제를 따라가야 한다.
- 자기소개 + 배경설명 + 실제 질문이 섞인 입력에서는 질문 clause를 우선한다.
- topic continuity는 보조 신호이지 현재 질문 초점을 덮어쓰면 안 된다.

## 8. 운영 규칙
새 기능을 핫패스에 넣기 전에 아래를 점검한다.
- 이 기능이 근거 일치를 높이는가?
- hallucination 가능성을 낮추는가?
- 현재 입력을 근거처럼 오인하게 만들지 않는가?
- 검색 후 재사고를 방해하지 않는가?

이 질문에 확신 있게 예라고 답하지 못하면, 핫패스에 넣지 않는다.
