# MACHI

MACHI는 단순한 챗봇 프로젝트가 아닙니다.

이 레포가 장기적으로 지향하는 것은, 한 사람을 장기적으로 보조하는 **로컬 Cognitive Partner**, 더 크게는 **개인 전용 인지 시스템**입니다.  
목표는 "그럴듯하게 답하는 AI"를 만드는 것이 아니라, **사용자와 세계를 구조적으로 이해하고, 장기 기억을 유지하고, 판단의 일관성을 보조하며, 필요할 때 스스로 구조를 재정렬할 수 있는 시스템**에 가까워집니다.

---

## 1. MACHI의 상위 목표

MACHI가 지향하는 상위 목표는 크게 다섯 가지입니다.

### 1-1. 개인 전용 Cognitive Partner

MACHI는 사용자를 위한 범용 비서라기보다, 특정 사용자를 장기적으로 보조하는 파트너를 목표로 합니다.

이 파트너는::

- 두 번째 관점
- 모순 탐지기
- 장기 일관성 검사기
- 중요도 증폭기

에 가깝고, 지휘자나 치료자, 도덕 권위가 되는 것을 목표로 하지 않습니다.

### 1-2. 구조 우선 이해

MACHI는 "말을 잘하는 것"보다 **구조를 이해하는 것**을 더 중요하게 봅니다.

핵심 문제의식은 다음과 같습니다.

- 결과만 맞는 답보다, 왜 그런지 드러나는 답이 더 중요하다.
- 사용자를 문장 몇 개로 평면화하지 않고, 작동 구조로 이해해야 한다.
- 감정적 지지는 중요하지만, 먼저 구조적 납득이 있어야 제대로 작동한다.

즉, MACHI는 정보 제공기가 아니라 **구조적 이해를 돕는 시스템**을 지향합니다.

### 1-3. 장기 기억과 사용자 모델

MACHI는 매 대화를 새로 시작하는 일회성 인터페이스가 아니라,
시간이 지날수록 더 안정적으로 사용자를 이해하는 시스템을 지향합니다.

여기서 중요한 것은 단순한 사실 저장이 아니라:

- 사용자 모델
- 반복 패턴
- correction 기록
- 장기 일관성
- 판단 기준의 변화

를 계속 추적하는 것입니다.

### 1-4. 자아 / 세계 / 기억 / 판단의 분리

MK1~MK3 문서에서 반복적으로 보이는 더 큰 방향성은,
AI를 하나의 거대한 블랙박스로 두는 대신 아래 층위들을 분리하려는 시도입니다.

- 세계를 이해하는 층
- 사용자를 이해하는 층
- 장기 기억을 관리하는 층
- 현재 의도와 판단을 다루는 층
- 최종 언어화 / 행동을 수행하는 층

즉, MACHI는 "하나의 프롬프트가 모든 걸 떠안는 구조"보다,
**분리되고 갱신 가능한 인지 구조**를 목표로 합니다.

### 1-5. AI 자체의 구조 재구성

MACHI는 단순히 기존 LLM 위에 기능을 덧붙이는 프로젝트로 끝나지 않습니다.

초기 문서들에는 다음과 같은 더 큰 목표가 드러납니다.

- 구조 자체가 학습되는 시스템
- 개념이 분화되고 재조직되는 시스템
- 자아 / 세계 / 분류가 분리된 시스템
- 기억 누적이 아니라 그래프 재작성과 재정렬이 일어나는 시스템

즉, MACHI는 장기적으로 **"더 나은 대화형 앱"** 보다 **"더 나은 인지 아키텍처"** 를 향하고 있습니다.

---

## 2. 무엇을 만들고 싶은가

한 문장으로 쓰면 이렇습니다.

> MACHI는 사용자를 장기적으로 이해하고, 구조적으로 사고를 보조하며, 기억과 판단을 재구성할 수 있는 로컬 개인 인지 시스템을 만들려는 프로젝트입니다.

좀 더 풀어 쓰면:

- 사용자의 판단 기준을 장기적으로 기억하고
- 새로운 evidence가 들어오면 기존 이해를 수정할 수 있으며
- 세계 지식과 사용자 모델을 구분해서 다루고
- 답변보다 이해, 위로보다 구조, 즉흥성보다 일관성을 우선하는

시스템을 만들고 싶어 합니다.

---

## 3. MACHI가 중요하게 보는 원칙

### 3-1. Local-first

이 프로젝트는 가능한 한 **기억과 구조의 소유권이 사용자에게 남는 로컬 시스템**을 지향합니다.

### 3-2. Structure before comfort

사용자에게 맞는 좋은 응답은 공허한 위로나 과한 친절보다,
먼저 구조를 제공하고 그 다음 온기를 더하는 방식이라고 봅니다.

### 3-3. Verification before performance

그럴듯한 말투보다:

- 사실 확인
- 논리 검증
- 전제 점검
- 불확실성 명시

를 우선합니다.

### 3-4. Memory is revisable

기억은 절대 진실 저장소가 아니라, 나중에 correction과 새 evidence로 다시 바뀔 수 있는 층이어야 합니다.

### 3-5. The user remains the decision owner

MACHI는 사용자를 대신 살아주는 시스템이 아니라,
사용자의 판단을 더 정교하게 만들고 장기 일관성을 보조하는 시스템이어야 합니다.

## 4. 버전별 흐름

이 레포는 하나의 완성품이 아니라, MACHI를 향해 가는 여러 단계의 실험 기록입니다.

### `MK1`

초기 인지 구조 아이디어와 Cognitive Partner 개념이 강하게 드러나는 단계입니다.

- `identity.md`, `patterns.md`, `wake_up_prompt.txt`
  - 사용자의 사고 방식, 관계 규칙, 파트너 역할 정의
- `idea_brainstorming/`
  - 그래프 기반 인지 구조, 모듈러 구조, 구조 학습, 자아/세계 분리 같은 더 상위 아키텍처 탐색

즉, **MACHI의 철학적/아키텍처적 출발점**에 가깝습니다.

### `MK2`

Cognitive Partner 개념을 더 운영 가능한 규칙으로 정리한 단계입니다.

- `system_core.md`
- `persona_support.md`
- `user_model.md`

여기서는 구조, 승인, 자율성, 사용자 모델, 응답 스타일 같은 **운영 규칙**이 더 선명해집니다.

### `MK3`

로컬 에이전트 형태가 더 구체화된 단계입니다.

- 로컬 모델
- trusted search
- 단순 프로필
- 최근 대화 메모리

즉, **실제 도구형 개인 AI**에 가까운 프로토타입 성격이 강합니다.

### `MK4`

현재 가장 적극적으로 다듬고 있는 memory 중심 프로젝트입니다.

MK4는 MACHI의 상위 목표 중 특히 아래를 구현하려고 합니다.

- evidence-first memory
- correction 가능한 memory
- topic 기반 의미 축
- general / candidate / confirmed tier
- chat / attachment / project artifact를 공통 기억 구조로 수렴시키는 흐름

즉, MK4는 **MACHI 전체 비전 중 "기억(memory) 계층"을 실제로 구현하는 현재 작업 축**에 가깝습니다.

이 레포는 하나의 단일 앱이라기보다 다음을 함께 담고 있습니다.

- 상위 철학 문서
- Cognitive Partner 설계 문서
- 인지 구조 아이디어 메모
- 로컬 AI 프로토타입

따라서 이 저장소를 읽을 때는:

1. 루트 `README.md`로 MACHI의 상위 목표를 먼저 이해하고
2. 각 MK 폴더를 버전/실험 단계로 보고
3. 현재 구현이 가장 많이 진전된 축은 `MK4`라고 이해하는 것이 가장 자연스럽습니다.

## 6. 지금 시점에서의 가장 중요한 해석

현재 MACHI를 가장 정확하게 표현하면 이렇습니다.

- 챗봇 프로젝트라기보다 **개인 인지 시스템 프로젝트**
- 프롬프트 실험이라기보다 **구조 실험**
- 개인화 UX라기보다 **장기적인 사용자 이해**
- 메모 앱이라기보다 **수정 가능한 기억 엔진**
- 답변 생성기라기보다 **사고 보조 파트너**

그리고 장기적으로는, 단순히 "잘 대답하는 모델"을 넘어서
**사용자와 세계를 함께 구조적으로 이해하는 AI 아키텍처**를 목표로 합니다.

## 8. 마무리

MACHI는 아직 완성된 제품이 아닙니다.

하지만 이 프로젝트가 묻고 있는 질문은 분명합니다.

- AI는 사용자를 어떻게 장기적으로 이해해야 하는가?
- 개인화는 프롬프트가 아니라 어떤 구조가 담당해야 하는가?
- 기억은 어떻게 수정 가능해야 하는가?
- 사용자의 판단을 대신하지 않으면서도, 어떻게 더 정확한 사고를 돕는가?
- 장기적으로는 LLM 위의 앱이 아니라, 어떤 인지 구조를 만들어야 하는가?

이 레포는 그 질문들에 대한 여러 단계의 답변을 담고 있습니다.
