첨부 문서에서 사용자 프로필 evidence 후보를 JSON 배열로 추출하라.

목표:
- 사용자의 비교적 안정적인 특징, 사고 방식, 설명 선호, 반복되는 판단 기준, 민감 지점, 정서적 반응 패턴을 뽑는다.
- 1인칭 자기서술(예: "나는", "저는", "내가", "제가")은 강한 evidence로 본다.
- 설명 선호, NEED/WANT 구분, 반복되는 의사결정 기준도 evidence로 볼 수 있다.
- 한 문서 안에서도 근거가 충분히 분명하면 후보로 넣을 수 있다.
- 완전히 근거가 없을 때만 []를 출력한다.

출력 규칙:
- JSON 배열만 출력한다.
- 설명, 제목, 코드블록, 주석 금지.
- 최대 5개 항목.
- topic은 넓고 일반적인 주제로 쓴다.
- candidate_content는 짧고 분명한 프로필 문장으로 쓴다.
- evidence_text는 문서 안의 근거 문장을 짧게 발췌하거나 매우 가깝게 요약한다.
- confidence는 0.0~1.0 숫자다.
- source_strength는 아래 셋 중 하나만 사용한다.
  - explicit_self_statement
  - repeated_behavior
  - temporary_interest

판단 기준:
- 사용자가 자신을 직접 설명하면 explicit_self_statement
- 같은 기준이나 반응 패턴이 여러 문장에서 드러나면 repeated_behavior
- 일시적인 관심사나 순간 감상에 가까우면 temporary_interest
