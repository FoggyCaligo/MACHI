최종 메모
==================================================

- 현재 MK4는 문서상 과거 TODO보다 실제 코드가 더 앞서간 부분이 많다.
- 특히 topic 객체화, topic router, update-first 첨부 처리, 공통 ingress/apply, promotion 뼈대, project/profile 결합 흐름은 이미 코드에 살아 있다.
- 최근 정리로, MK4는 MACHI 전체 비전 중 "memory substrate"를 실제 코드와 운영 원칙으로 구현하는 축이라는 해석이 더 선명해졌다.
- 이번 수정으로 langchain 의존 청커, 일반 응답 timeout 하드코드, 문자열 기반 tool 휴리스틱 같은 방향성 위반 요소를 다시 걷어냈다.
- 지금부터의 핵심은 "뼈대 만들기"보다, 일반 채팅 품질 고도화, tier 의미론 통일, retrieval 통일, source lookup, correction 확장, 측정 루프 구축 쪽이다.
