문서와 현재 코드가 엇갈리는 대표 사례
==================================================

1. "topic 분류는 아직 규칙 기반"
- 현재 로컬 코드 기준으로는 과거 상태에 가깝다.
- 지금은 TopicRouter 중심이고, 임베딩 기반 keep/attach/create가 들어와 있다.

2. "project retrieval은 lexical scoring 기반"
- 이것도 현재 로컬 코드 기준으로는 과거 상태에 가깝다.
- 지금은 임베딩 기반 검색으로 바뀌어 있다.

3. "TopicRouter.resolve 인자 불일치"
- 현재 로컬에서는 이미 해결된 상태다.

4. "profile evidence는 아직 candidate까지만 반영"
- 현재 코드와 완전히 일치하지 않는다.
- 일부 경로에서는 confirmed까지 가는 조건부 승격이 이미 들어가 있다.
- 다만 채팅 경로는 아직 부족하다.

==================================================
최종 요약
==================================================

지금 MK4는
- topic / project 축 분리
- artifact → evidence → sync
- 운영 원칙 중심 prompt
- 공통 ingress/apply 뼈대

까지는 꽤 많이 왔다.

실제로 남은 핵심은
- 하드코딩 제거
- 일반 채팅을 첨부/프로젝트 수준으로 끌어올리기
- 일반 / 후보 / 확정 경계 통일
- retrieval 철학 통일
- 속도/측정/lookup 구조 만들기

이 다섯 축이라고 보는 게 가장 정확하다.
